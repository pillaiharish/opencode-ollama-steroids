# Code of Conduct

## Our Standards

This project expects professional, respectful collaboration focused on making the workflow safer and easier to reuse.

Examples of good participation:

- giving specific technical feedback;
- reporting security or privacy risks carefully;
- keeping examples generic and public-safe;
- respecting scope and review boundaries.

Unacceptable behavior includes harassment, personal attacks, publishing private data, or intentionally adding secrets, private prompts, private logs, or identifying information to public materials.

## Enforcement

Maintainers may remove comments, issues, pull requests, or contributors that violate these standards. Security and privacy issues may be handled privately before public discussion.
